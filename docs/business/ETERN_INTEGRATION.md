# ETERN INTEGRATION

## Purpose

ClayTube is a standalone product.

ETERN Labs is the product showcase and inquiry platform.

These must remain separate but connected.

---

## Relationship

```text
ClayTube (product)
    ↓
open-source GitHub repository
    ↓
public examples
    ↓
ETERN Labs
    ↓
service inquiry
```

---

## Repository Separation

### ClayTube Repository

Purpose:

- CLI development
- documentation
- issue tracking
- releases
- example templates

Example:

```text
github.com/cable8mm/claytube
```

---

### ETERN Labs Repository

Purpose:

- showcase products
- present examples
- explain capabilities
- collect inquiries

Example:

```text
labs.etern.co.kr
```

---

## Strategic Rule

ClayTube must remain product-focused.

ETERN Labs must remain service-focused.

Do not mix:

- product documentation
- sales messaging

Keep separation clear.

---

## Long-Term Goal

Build multiple small digital products.

ClayTube is the first product.

ETERN Labs becomes the central place where all products are:

- introduced
- demonstrated
- sold
- supported
